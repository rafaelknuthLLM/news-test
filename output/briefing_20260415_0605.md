## Top Signals This Week

### 1. Anthropic Deprecates Claude Sonnet and Opus 4 -- A Model Generation Is Turning Over

The biggest signal this week is quiet but unmistakable: the `anthropic` Python SDK v0.95.0 (released April 14) explicitly marks **Sonnet 4 and Opus 4 as deprecated**. Meanwhile, SDK v0.90.0 (April 7) quietly added support for a model called `claude-mythos-preview`.

Read together, this tells one story: Anthropic is rotating to a new generation. "Mythos" appears to be an internal codename for what comes after the Claude 4 family. Deprecating the current flagship models in the SDK changelog -- not just removing them from a pricing page -- is how you signal to developers to start planning migrations.

For builders: if you hardcode `claude-opus-4` or `claude-sonnet-4` model IDs anywhere, you have a migration coming. Watch the Anthropic changelog carefully over the next 2--4 weeks.

### 2. Claude Managed Agents Lands in the Official SDK

SDK v0.92.0 (April 8) added "support for Claude Managed Agents" as a top-level feature. This isn't just a wrapper -- it's Anthropic taking multi-agent orchestration from a pattern you build yourself to a first-class API primitive.

A day later, Claude Code v2.1.101 got a parallel upgrade: the model can now "discover and invoke built-in slash commands like /init, /review, and /security-review via the Skill tool." What that means in practice is that Claude can now recursively call structured capabilities without human routing -- the agent SDK and the developer tool are converging.

This is Jensen Huang's "Applications" layer getting a new foundation. Developers who've been stitching together multi-agent systems manually now have a vendor-supported path.

### 3. Claude Code Is Shipping at Abnormal Velocity

Nine releases in seven days. That pace is unusual even for a fast-moving CLI tool. The substantive changes cluster into three themes:

**Prompt caching gets enterprise-grade:** v2.1.108 added `ENABLE_PROMPT_CACHING_1H` -- a 1-hour cache TTL for API key, Bedrock, Vertex, and Foundry users. The previous default was 5 minutes. For long-running agentic sessions, this is a meaningful cost reduction.

**Session continuity improves:** A new `/recap` command provides context when returning to a session. Combined with fixes to `--resume` for large sessions and improved `/resume` picker defaults, Anthropic is clearly responding to users who leave and re-enter long-running work sessions.

**Security hardening:** v2.1.98 fixed a Bash tool permission bypass where backslash-escaped flags could be auto-allowed as read-only and lead to arbitrary code execution. v2.1.101 fixed a command injection vulnerability in the POSIX `which` fallback. When security vulnerabilities are showing up in release notes of a developer tool, it means real-world adversarial use is being probed.

### 4. Gemma4 Is Breaking Things Fast -- Which Means It's Spreading Fast

HuggingFace Transformers shipped four patch releases (v5.5.1 through v5.5.4) in under 12 days, almost entirely focused on **Gemma4**. The bugs being fixed -- device map failures, KV state sharing errors, tokenizer regressions on Kimi-K2.5 -- are exactly the bugs you see when a model goes from "just released" to "running in real production pipelines" at scale.

The v5.5.0 release itself (April 2) added Gemma4 with a notably novel vision architecture: instead of squashing images to a fixed square, it preserves natural aspect ratios within a configurable token budget (70 to 1,120 soft tokens per image). That flexibility is likely driving adoption for multimodal use cases, and the adoption is surfacing edge cases faster than the initial release could catch.

For the Models layer: Gemma4's architecture choices (2D RoPE, spatial positional encoding, configurable token budget) are worth tracking as potential patterns for the next generation of vision-language models broadly.

The `huggingface/transformers-pytorch-gpu` image stagnating at 447K and not updating since October 2025 is a quiet signal that HuggingFace's Docker distribution strategy may be losing ground to Ollama's simpler packaging.

## Worth Watching

**1. The "claude-mythos-preview" model.** It's in the SDK, Sonnet/Opus 4 are deprecated, and no public announcement has dropped yet. Either this week or next, Anthropic is expected to name and launch whatever comes after Claude 4. The SDK is already wired.

**2. Managed Agents adoption patterns.** Now that multi-agent orchestration is a first-class API, watch whether the agentic framework ecosystem (CrewAI, AutoGen, LlamaIndex) pivots to using it as a backend or builds competing abstractions. The version cadence of crewai (1.14.1), autogen (0.11.5), and llama-index (0.14.20) over the next two weeks will be a leading indicator.

**3. Gemma4 tokenizer stability.** The Kimi-K2.5 tokenizer regression fixed in v5.5.4 is a warning sign -- adding a new model architecture can break existing tokenizer behavior in subtle ways. If more regressions appear for other models (Qwen2.5, Mistral 4) in the next patch releases, it suggests the v5.5.x series needs a bigger refactor before it stabilizes.
