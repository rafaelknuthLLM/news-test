**v2.1.108 -- April 14, 2026**
https://github.com/anthropics/claude-code/releases/tag/v2.1.108
(deterministic) 24 changelog items. 48 total reactions: 22 👍 8 🎉 5 😄 5 ❤️ 4 🚀 4 👀 0 👎. Positive ratio: 81.2%.

What changed -- the highlights:

**The recap feature.** When you come back to a long coding session after being away, Claude can now give you a "here's where we left off" summary. You can trigger it manually with `/recap` or configure it to happen automatically. This is the equivalent of a colleague saying "okay, so when you stepped out, we'd just figured out the database problem and were about to tackle the login flow."

(observation -- tutor agent) This one genuinely delights me. Long AI coding sessions are messy -- you go deep into a problem, get interrupted, come back hours later, and the context is gone from your head even if it isn't gone from the tool. The recap is essentially a memory prosthetic. Small feature, real quality-of-life impact.

**Prompt caching -- what it is and why it matters.** There are now two new settings: one to keep the "prompt cache" for a full hour, one to force it to only 5 minutes. The prompt cache is like a scratchpad the AI keeps between messages in a session -- without it, every message has to re-read everything from scratch, which is slower and costs more. An hour-long cache means the AI doesn't have to reload your entire codebase every few minutes.

(observation -- tutor agent) The detail that subscribers who turned off telemetry were silently getting the *short* cache instead of the *long* one -- that's the kind of bug that would be completely invisible to users and quietly make the tool worse for privacy-conscious people. Worth fixing.

**Slash commands can now discover themselves.** Claude can now find and use built-in commands like `/init` or `/security-review` on its own, through something called the Skill tool. Think of slash commands as shortcuts -- `/review` means "review my code," `/security-review` means "check for vulnerabilities." Previously Claude might not have known these shortcuts existed. Now it can look them up and use them autonomously.

**Error messages got smarter.** Rate limits (you're sending too many requests) now look different from plan limits (you've used up your monthly quota). Previously they probably looked the same, which would be maddening -- the fix for one is "wait a minute," the fix for the other is "upgrade your plan."

**Memory footprint reduction.** The tool now loads language-specific highlighting rules only when it needs them, rather than loading everything at startup. Like a library that keeps most books in storage and only pulls them when someone asks.

(observation -- tutor agent) The bug about diacritical marks -- accents, umlauts, cedillas -- being *dropped from responses* when a language setting was configured is quietly significant. If you're working in French, German, Portuguese, or dozens of other languages and your language setting was on, Claude was silently mangling its own output. That's a bad one. The fix is listed without fanfare, which I find charming in a workmanlike way.

**v2.1.105 -- April 13, 2026**
https://github.com/anthropics/claude-code/releases/tag/v2.1.105
(deterministic) 37 changelog items. 68 total reactions: 27 👍 11 🎉 8 😄 8 ❤️ 8 🚀 6 👀 0 👎. Positive ratio: 79.4%.

This is the big one in the batch. Let me walk through what I found interesting.

**Hooks can now block compaction.** Compaction is what happens when a conversation gets so long that the tool needs to compress it to save space -- like summarizing old chapters of a book to make room for new ones. A "hook" is a piece of custom code that runs automatically when something happens. Now you can write a hook that says "wait, don't compress yet" -- maybe because you need the full history intact for compliance, or because you're mid-operation and compacting right now would break something.

(observation -- tutor agent) The specific mechanism -- exiting with code 2 or returning a structured block decision -- is very developer-brained. But the concept is: you can now veto the tool's housekeeping at a critical moment. That's meaningful control for power users.

**Background monitors for plugins.** Plugins can now have a piece of themselves running quietly in the background from the moment a session starts, watching for things. Think of it like a smoke detector that arms itself when you enter a room, versus one you have to manually turn on.

**WebFetch strips CSS and JavaScript from web pages.** When Claude fetches a webpage to read it, it now throws away the styling code and interactive scripts -- the stuff that makes pages look pretty -- and keeps only the actual text content. A modern webpage can be 90% visual machinery and 10% words. This was apparently filling up Claude's reading budget before it even got to the article.

(observation -- tutor agent) This one surprised me. It implies that before this fix, Claude could fetch a webpage, spend most of its "reading budget" on CSS rules describing button colors, and have almost nothing left for the actual content. That's a funny failure mode -- like summarizing a restaurant menu but spending all your notepad space on the font choices.

**The `/doctor` command got a fix button.** `/doctor` checks the health of your installation. Now it has status icons, and you can press `f` to have Claude automatically fix whatever it finds wrong. Self-healing tooling.

**MCP output truncation now gives format-specific advice.** MCP -- Model Context Protocol -- is a standard for connecting Claude to external tools and data sources. When those tools return too much data, Claude has to truncate it. Now instead of a generic "this was too long," it gives specific advice: "if this is JSON, try `jq`; if it's text, here's how to read it in chunks." Like getting directions instead of "you're too far away."

**The color bug for SSH users.** If you're connecting to a server remotely and using certain modern terminal apps, the colors were washed out. This is surprisingly fiddly to get right -- different terminal emulators report their capabilities differently, and SSH adds another layer of ambiguity. Six specific terminal apps are named in the fix.

(observation -- tutor agent) 37 items in a single release, with zero negative reactions. This is a "we've been collecting bugs and we are clearing the backlog" release. The breadth is almost comedic -- diacritical marks, color palettes, ASCII art, Python logging libraries, scheduled tasks firing twice, survey shortcut keys triggering mid-sentence. This team is clearly getting detailed bug reports from a wide and active user base.
