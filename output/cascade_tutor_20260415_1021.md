## v2.1.105 -- April 13, 21:53 UTC

**URL:** https://github.com/anthropics/claude-code/releases/tag/v2.1.105

**Changelog:** 37 bullet items across 38 lines. (deterministic)

**Reactions:** 68 total (deterministic). 54 positive, 0 negative, 14 neutral. Positive ratio: 79.4%. (deterministic)

This is the meatiest release in the set -- 37 items in one drop, covering new features, improvements, and a long list of bug fixes. Let me walk through the interesting ones.

**New things:**

- **EnterWorktree path parameter** -- A "worktree" in coding terms is like having multiple parallel workspaces for the same project. Imagine a folder on your desk for the current version of your project, and a second folder for an experimental branch you're testing. This lets Claude switch into an existing one rather than always creating new ones.

- **PreCompact hook** -- "Compaction" is what happens when a conversation gets very long and the tool needs to summarize or compress earlier parts to make room for new ones (AI models can only hold so much in their working memory at once, like a notepad with limited pages). A "hook" is a way for users or plugins to inject their own code at specific moments -- in this case, *before* that compression happens. The new feature lets a hook *block* the compaction entirely by sending a specific signal. That's a meaningful amount of control handed to advanced users. (observation -- tutor agent)

- **Background monitor support for plugins** -- Plugins can now run quietly in the background watching for things, rather than only responding when asked. Think of it like the difference between a smoke alarm (always watching) versus a fire extinguisher (only used when you reach for it).

- **/proactive is now an alias for /loop** -- Slash commands (the `/init`, `/help`, `/model` style instructions) are how users talk to the tool directly. An alias just means two names for the same thing. Small convenience.

**The WebFetch improvement catches my attention:** the tool now strips all visual styling code (`<style>` and `<script>` tags) from web pages before reading them, so it doesn't waste its limited reading capacity on CSS rules that tell a page what color to be. That's a quietly smart fix -- it means the AI gets more actual text per page fetch. (observation -- tutor agent)

**MCP** shows up twice here. MCP is a protocol -- think of it as a standardized language that lets Claude talk to external tools and data sources. It gets mentioned a lot across these releases, which suggests it's a live, actively-used part of the system. (observation -- tutor agent)

The bug fix list is long and specific -- garbled terminal output, paste not working in the right places, a color palette problem on certain terminal apps over remote connections. The level of detail suggests real user reports being systematically worked through. (observation -- tutor agent)

## v2.1.108 -- April 14, 19:12 UTC

**URL:** https://github.com/anthropics/claude-code/releases/tag/v2.1.108

**Changelog:** 24 bullet items. (deterministic)

**Reactions:** 48 total (deterministic). 39 positive, 0 negative, 9 neutral. Positive ratio: 81.2%. (deterministic)

A substantial release. The standouts:

**Prompt cache TTL controls** -- "Prompt caching" is a performance trick: rather than re-reading your entire conversation from scratch every time Claude responds, the system caches (temporarily stores) parts of it. TTL means "time to live" -- how long that stored copy is kept before it expires. A 1-hour TTL means the cache lasts an hour; 5-minute means it refreshes more frequently. The new environment variables (`ENABLE_PROMPT_CACHING_1H`, `FORCE_PROMPT_CACHING_5M`) let users control this directly. This matters for cost and speed, especially on large projects. The old variable for the same thing on Amazon's Bedrock platform is now deprecated -- meaning it still works but is officially being phased out.

**Recap feature** -- When you come back to a coding session after being away, the tool can now give you a summary of where you left off. Like a "previously on..." at the start of a TV episode. Configurable, which is thoughtful -- not everyone wants this. (observation -- tutor agent)

**Slash command discovery via Skill tool** -- The model can now *find and use* its own built-in commands (`/init`, `/review`, `/security-review`) by itself, rather than only responding when a human types them. This is the tool becoming slightly more self-directing. (observation -- tutor agent)

**The /model warning** is interesting from a technical standpoint: switching AI models mid-conversation means the tool has to re-read your entire history from scratch, *uncached*, which costs more and takes longer. The warning is honest about that tradeoff in a way the tool apparently wasn't before. (observation -- tutor agent)

**The diacritical marks fix** -- accents, umlauts, cedillas (the marks under a ç, over an é, through an ñ) were being dropped from responses when the user had a non-English language configured. That's a real internationalization bug -- the kind that's invisible to English-only users but frustrating for everyone else. (observation -- tutor agent)
